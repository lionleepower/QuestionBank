# Python 问题与批改记录

这个文件专门记录 Python 学习时遇到的问题、自己的回答、批改反馈和最后要记住的结论。

目前这个文件先作为 Python 题库入口和模板使用。真正的 Python 问题以后新增到这里；Bash、shell、Slurm、awk、benchmark 脚本相关内容统一放在 `01_bash.md`。

## 为什么要这样整理？

之前 `02_python.md` 里大部分内容其实是 Bash 题目，和 `01_bash.md` 重复。这样会带来两个问题：

- 随机抽题时同一批 Bash 问题会重复出现。
- 文件名是 Python，但内容是 Bash，后续复习时很难按主题定位。

整理后的规则是：

- `01_bash.md`：Bash、shell、Slurm、awk、benchmark 脚本。
- `02_python.md`：Python 语法、数据结构、函数、模块、脚本、调试、第三方库。

## 推荐问题模板

以后新增 Python 问题时，建议每道题都使用下面这个结构。这样 `scripts/quiz.py` 可以稳定地把“题目”和“答案”分开。

~~~markdown
## YYYY-MM-DD：问题标题

**我的问题**

这里写你当时真正卡住的问题。尽量写成可以直接拿来测试自己的问句。

**上下文 / 代码**

```python
# 如果问题和代码有关，把最小例子放在这里
```

**我的理解 / 回答**

这里先写你自己的想法。就算不确定也要写，因为之后批改才知道错在哪里。

**批改**

这里写纠正、补充解释、正确写法，以及为什么。

**记住**

```text
把这道题压缩成 1-3 条以后必须记住的结论。
```
~~~

## 抽题友好的写法

为了让随机抽题更好用，每道题尽量满足：

- 标题用 `## 日期：问题标题`。
- `我的问题` 写清楚要回答什么。
- 代码放在 `上下文 / 代码`，不要只把代码塞进标题。
- `我的理解 / 回答`、`批改`、`记住` 放在后面，作为答案文件内容。
- 一道题只问一个核心概念；如果有多个混在一起，就拆成多道题。

## 示例

~~~markdown
## YYYY-MM-DD：Python 里的 list 和 tuple 有什么区别？

**我的问题**

Python 里的 `list` 和 `tuple` 都能保存多个元素，它们最重要的区别是什么？

**上下文 / 代码**

```python
nums = [1, 2, 3]
point = (10, 20)
```

**我的理解 / 回答**

`list` 可以修改，`tuple` 不能修改。

**批改**

方向正确。更准确地说，`list` 是可变序列，适合保存之后还会增删改的数据；`tuple` 是不可变序列，适合表达固定结构的数据。

**记住**

```text
list = 可变；tuple = 不可变。
会变化的数据用 list；固定结构的数据可用 tuple。
```
~~~
